🚀 Git Practice Missions (Task-Only Version)
🟢 LEVEL 1: Local Basics

Goal: Understand tracking & commits

Create a new project folder and initialize it as a Git repository
Create a file and write some basic content
Save it as your first version
Modify the file by adding more content
Check what changed before saving again
Save the updated version
View the history of changes
🟡 LEVEL 2: Remote Connection (GitHub)

Goal: Connect local ↔ remote

Create an empty repository on GitHub
Link your local project to that remote repository
Upload your project to GitHub
Make a new change locally
Upload the new change again
Verify on GitHub that both versions exist
🔵 LEVEL 3: Branching Workflow

Goal: Work with multiple branches

Create a new branch for a feature
Switch to that branch
Add a new file related to that feature
Save your changes in that branch
Switch back to the main branch
Confirm that the new file is not present here
Merge the feature branch into the main branch
Verify the file now appears in main
🔴 LEVEL 4: Merge Conflict (Important 🔥)

Goal: Experience and resolve conflict

Create a new branch from main
In that branch, modify a specific line in a file
Save the change
Switch back to main
Modify the same line in a different way
Save the change
Try to merge the branch into main

👉 You should now hit a conflict

Open the file and inspect conflict markers
Manually decide what the final content should be
Save and finalize the merge
🟣 LEVEL 5: Undo & Recovery

Goal: Learn reset & revert

Make 3–4 consecutive changes and save each
View the full history
Go back to an older version but keep your changes
Try going back and completely discarding changes
Undo a specific commit without deleting history
Observe how history looks after this
🟤 LEVEL 6: Stash Workflow

Goal: Handle unfinished work

Start editing a file but don’t save it yet
Temporarily store this unfinished work
Switch context (like moving to another task/branch)
Come back and restore your saved work
Continue and save it properly
⚫ LEVEL 7: Real-World Simulation

Goal: Think like a developer in a team

Create two different feature branches
Make different changes in both
Merge one branch into main
Then merge the second branch
Handle any conflicts if they arise
Push everything to GitHub
